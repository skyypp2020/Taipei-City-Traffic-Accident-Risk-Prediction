## 1. 這個專案在做什麼？

想像你是台北市的交通局長。每年有超過 2 萬件交通事故，你想知道：

- **哪些路口最危險？**（找熱點）
- **這些危險路口，有沒有裝照相設備？**（找缺口）
- **未來還會有多少事故？**（預測趨勢）
- **應該在哪裡優先裝照相設備？**（給建議）

這個分析就是幫你自動回答這四個問題。

輸入：2021–2025 年的交通事故資料（政府公開）
輸出：熱點地圖、事故預測圖、設備設置建議清單

## Step 1：安裝 Python

Python 就是這個分析用的程式語言，就像 Excel 是微軟的工具，Python 是資料分析的工具。

### 1-1. 下載 Python

1. 打開瀏覽器，前往：**https://www.python.org/downloads/**
2. 點擊最大的黃色按鈕「Download Python 3.xx.x」
3. 下載後打開安裝程式

### 1-2. 安裝時的重要設定

安裝程式出現後，**一定要勾選最下面的選項**：

```
☑ Add Python to PATH
```

（這個選項讓電腦找得到 Python，沒勾的話後面會出問題）

然後點「Install Now」就好了。

### 1-3. 確認安裝成功

安裝完後，按 `Windows 鍵 + R`，輸入 `cmd`，按 Enter，打開黑色視窗（命令提示字元）。

輸入以下指令，按 Enter：

```
python --version
```

如果看到類似 `Python 3.11.x` 的回應，就成功了 ✓

## Step 4：建立分析環境

這步驟是安裝分析需要用到的「工具套件」，就像你裝 Office 需要安裝 Word、Excel 一樣，Python 的工具也要個別安裝。

### 4-1. 打開命令提示字元，移動到專案資料夾

按 `Windows 鍵 + R`，輸入 `cmd`，按 Enter。

然後輸入（把路徑換成你放專案的地方）：

```
cd C:\Users\你的名字\Desktop\Taipei-City-Traffic-Accident-Risk-Prediction
```

> 💡 小技巧：在 Windows 資源管理員裡，對著資料夾按住 `Shift` 再右鍵，選「在此開啟命令提示字元」，可以省去輸入路徑的麻煩。

### 4-2. 建立「虛擬環境」

虛擬環境就是一個獨立的空間，讓這個專案的工具不會跟你電腦上其他程式衝突。

```
python -m venv venv
```

執行後你會看到多了一個 `venv` 資料夾，這很正常。

### 4-3. 啟動虛擬環境

```
venv\Scripts\activate
```

成功後，命令提示字元最左邊會出現 `(venv)` 的字樣，像這樣：

```
(venv) C:\Users\你的名字\Desktop\Taipei-...>
```

### 4-4. 安裝所有需要的工具

```
pip install -r requirements.txt
```

這個指令會自動安裝所有分析需要的工具套件，大概需要 3–5 分鐘，等它跑完就好。

### 4-5. 確認安裝成功

```
python -c "import pandas, numpy, sklearn, xgboost, folium; print('全部安裝成功')"
```

看到「全部安裝成功」就 OK 了。

### 📦 s01：資料清理

**在做什麼：** 讀取 5 份事故資料，刪掉有問題的紀錄（例如座標是空白的），整理好格式。

```
python src/s01_clean.py
```

**跑完後你會看到：**
```
清理完成，保留 118,980 筆
```

**產生的檔案：**
- `data/processed/accidents.parquet`：清理好的事故主檔
- `outputs/anomalies.csv`：被刪掉的 3 筆問題資料（附原因說明）

> 📝 這 3 筆被刪的原因都是「座標欄位是空的」，沒有辦法知道事故發生在哪裡，所以刪掉。

---

### 🗺️ s02：找出事故熱點

**在做什麼：** 用一個叫 DBSCAN 的算法，找出事故集中的地方（熱點）。

**DBSCAN 的概念：** 想像你把 118,980 個事故地點全部畫在地圖上。DBSCAN 會自動找出「這裡的點特別密集」的區域，把它框起來叫做熱點。我們設定：
- 半徑 50 公尺內
- 至少要有 80 件事故
才算是一個熱點。

```
python src/s02_hotspot.py
```

**跑完後你會看到：**
```
DBSCAN 完成：145 個熱點，熱點內事故 22,858 筆
熱點內占比：19.2%
```

**產生的檔案：**
- `data/processed/hotspots.csv`：145 個熱點的詳細資料

---

### 📅 s03：整理時間序列

**在做什麼：** 把事故資料按時間整理，做成三份時間表格。

**時間序列是什麼？** 就是「某個時間點發生了幾件事」的紀錄，像股票的每日收盤價、氣溫的每日記錄，都是時間序列。我們做的是事故的每日件數。

```
python src/s03_series.py
```

**跑完後：**
- `daily_series.csv`：1,826 天（5 年）每天的全市事故數
- `hotspot_monthly.csv`：145 個熱點 × 60 個月的事故數表格
- `dist_vectors.csv`：每個熱點「幾點、星期幾、幾月」的事故分布比例

---

### 🔧 s04：準備預測用的特徵

**在做什麼：** 把時間序列轉換成預測模型看得懂的格式，例如「昨天有多少件？7 天前有多少件？這是星期幾？幾月份？」

**為什麼要這樣做？** 機器學習模型（後面的 XGBoost）不懂「日期」，它只懂數字。我們要把「2021 年 3 月 15 日，星期一」轉換成「lag_7 = 68、ma_7 = 71.3、weekday_0 = 1（是星期一）、month = 3」這樣的格式。

**防止偷看未來：** 所有特徵都設計成只能使用「現在之前」的資訊，不會不小心用到未來的資料（這在時序預測中是個常見的錯誤）。

```
python src/s04_features.py
```

**跑完後：**
- `data/processed/feature_matrix.csv`：1,798 天 × 18 個特徵的表格
- `outputs/feature_definitions.csv`：每個特徵的說明

---

### 🔍 s05：熱點分類

**在做什麼：** 分析 145 個熱點各自的「事故時間規律」，把它們分成幾個型態。

**怎麼分類？** 先用 PCA（降維）把複雜的時間分布壓縮成幾個主要方向，再用 K-means 聚類找出相似的熱點群組。

```
python src/s05_pca_kmeans.py
```

**跑完後你會看到：**
```
型態分布：
  全日型：60 個熱點
  通勤尖峰型：40 個熱點
  假日型：25 個熱點
  夜間型：20 個熱點
```

**這些型態代表什麼：**

| 型態 | 說明 | 建議設備 |
|---|---|---|
| 全日型 | 一整天事故分布比較均勻 | 綜合評估 |
| 通勤尖峰型 | 早上 7–9 點、下午 5–7 點特別多 | 闖紅燈照相 |
| 假日型 | 相較其他群週末比例偏高 | 行人安全設備 |
| 夜間型 | 深夜 10 點–凌晨 3 點比例偏高 | 測速照相 |

---

### 📈 s06：預測未來事故數

**在做什麼：** 訓練三個預測模型，預測 2025 年每天的事故數，然後比較哪個模型最準。

**訓練資料：** 2021–2024 年（4 年）
**考試資料：** 2025 年（這些資料模型訓練時沒看過）

```
python src/s06_forecast.py
```

這個步驟最慢，大約需要 **20–30 秒**（因為要搜尋最好的 SARIMA 參數）。

**跑完後你會看到三個模型的成績：**

```
Naive(lag-7)   MAE=12.4 件（用一週前來猜，基準值）
SARIMA         MAE=9.2 件（統計方法，改善 26%）
XGBoost        MAE=9.3 件（機器學習，改善 25%）
```

**MAE 是什麼意思？** Mean Absolute Error，平均絕對誤差。MAE = 9.2 的意思是：模型預測「今天有 X 件事故」，平均跟實際差了 9.2 件。日均 62 件的情況下，誤差約 ±15%。

## 常見問題 FAQ

**Q：出現 `ModuleNotFoundError: No module named 'xxx'`**

代表某個套件沒裝到。解法：
```
pip install xxx
```
把 `xxx` 換成錯誤訊息裡出現的套件名稱。

---

**Q：出現 `FileNotFoundError`（找不到檔案）**

可能是資料沒有放對位置。確認 CSV 檔案在 `data/raw/` 資料夾裡，而且檔名完全正確（包含空格）。

---

**Q：`(venv)` 不見了**

關掉命令提示字元又開的話，需要重新啟動虛擬環境：
```
venv\Scripts\activate
```

---

**Q：s06 跑很久沒有反應**

SARIMA 搜尋最佳參數本來就需要 20–30 秒，等它就好，沒有壞掉。

---

**Q：我想換別的城市的資料可以嗎？**

可以，但需要修改：
- `src/config.py` 裡的座標範圍（`LON_RANGE`、`LAT_RANGE`）
- 重新確認資料的欄位格式

---

**Q：圖表上的中文是亂碼（豆腐字）**

在 Mac 或 Linux 上需要改字型設定。打開 `src/s08_visualize.py`，找到這行：
```python
plt.rcParams["font.family"] = ["Microsoft JhengHei", ...]
```
改成你系統上有的中文字型名稱（Mac 通常是 `PingFang TC`）。

---

**Q：我可以用這份分析做其他城市的研究嗎？**

當然可以，這是開源的分析框架。只要找到對應城市的事故開放資料，修改設定檔就能用。
