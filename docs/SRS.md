## 1. 簡介(Introduction)

### 1.1 目的(Purpose)
本文件定義「臺北市道路交通事故熱點風險預測與照相設備設置建議系統」之軟體需求,作為開發、測試與驗收之依據。閱讀對象為專題組員(開發者)與授課教師(驗收者)。

### 1.2 範圍(Scope)
本系統為批次執行之資料分析管線,輸入臺北市政府開放資料(110–114 年交通事故斑點圖、固定式照相設備一覽表)與第二階段外部資料(天氣、辦公日曆、捷運運量等),輸出:

1. 事故空間熱點清單與熱點型態分群結果
2. 全市日事故數與熱點月事故數之時間序列預測模型及其驗證指標(MAE、RMSE)
3. 照相設備建議設置清單(地點、排序依據、建議設備類型)
4. 報告所需之視覺化圖表

**不在範圍內**:因果效果估計(設置前後比較)——因照相設備資料無設置日期;即時資料介接(VD 車輛偵測器);Web 服務或使用者介面。

### 1.3 名詞定義(Definitions)

| 名詞 | 定義 |
|---|---|
| A1 事故 | 造成人員當場或 24 小時內死亡之交通事故(處理別 = 1) |
| A2 事故 | 造成人員受傷之交通事故(處理別 = 2) |
| 熱點(Hotspot) | DBSCAN 分群所得之事故空間密集區(eps = 50 m、min_samples = 80) |
| 覆蓋缺口(Gap) | 屬熱點且 300 m 內無任何固定照相設備之地點 |
| 時間指紋 | 熱點之 24 小時 + 一週七天 + 12 月份事故分布向量(43 維) |
| 80/20 切分 | 訓練期 2021/01/01–2024/12/31、驗證期 2025/01/01–2025/12/31 |

## 2. 整體描述(Overall Description)

### 2.1 產品視角(Product Perspective)
獨立執行之 Python 分析管線,無外部系統相依;以本機檔案系統為輸入輸出介面。系統處理流程對應課程之大數據分析流程:問題定義 → 資料收集 → 資料清理 → 特徵化 → 降維 → 分類與分群 → 建模 → 報告。

### 2.2 產品功能摘要(Product Functions)
1. 資料載入、編碼處理與清理合併(五年斑點圖 + 設備一覽表)
2. DBSCAN 空間熱點辨識
3. 時間序列化(全市日序列、熱點月序列、熱點時間分布向量)
4. 特徵萃取(滯後、移動統計、週期特徵;第二階段加入外部特徵)
5. PCA 降維與 K-means 熱點型態分群
6. 時間序列預測建模與 80/20 驗證
7. 覆蓋缺口分析與建議設置清單產出
8. 視覺化圖表產出

### 2.3 使用者特性(User Characteristics)
專題組員,具 Python 與資料分析基礎;以命令列依序執行各模組腳本。

### 2.4 限制(Constraints)
- C-1:程式語言限 Python 3.10+,使用開源套件
- C-2:資料僅限政府開放資料,報告須載明資料集名稱與下載位置(課程硬性要求)
- C-3:模型驗證必須採時間序列前 80% 訓練、後 20% 驗證,不得隨機切分
- C-4:全管線(LSTM 除外)於一般筆電 10 分鐘內可完成
- C-5:照相設備資料無設置日期,所有產出不得宣稱因果效果

### 2.5 假設與相依(Assumptions and Dependencies)
- A-1:斑點圖座標為 WGS84 經緯度且各年度欄位結構一致
- A-2:設備一覽表為現況快照,假設設備於分析期間多數已存在(用於覆蓋分析,非因果分析)
- A-3:第二階段外部資料可自公開平臺下載,且可以日期為鍵與日序列合併
- A-4:2021/5/19–7/26 COVID 三級警戒造成事故數結構性下降,須以虛擬變數控制

### 3.2 功能需求(Functional Requirements)

> 格式:每項需求含描述、輸入/輸出、驗收條件(AC)。優先度:M = Must、S = Should。

**FR-01 資料載入與清理(M)**
- 描述:以 cp950(`encoding_errors='replace'`)讀取 IN-1~IN-5 並縱向合併;統一欄名為「發生時間、處理別、肇事地點、X、Y」;剔除座標缺漏與超出範圍(經度 121.4–121.7、緯度 24.9–25.25)之紀錄;解析發生時間為 datetime,失敗者剔除;以 Big5 讀取 IN-6。
- AC-01a:清理後主檔 = 118,980 筆(合併 118,983 − 剔除 3)。
- AC-01b:清理 log 記錄各步驟剔除筆數。

**FR-02 空間熱點辨識(M)**
- 描述:對事故座標(轉弧度)執行 DBSCAN(metric=haversine、eps=50 m、min_samples=80,參數可自 config 調整);計算各熱點中心(座標平均)、事故總數、A1 數、代表地點(肇事地點眾數)。
- AC-02a:以預設參數重現 145 個熱點、熱點內事故占比 19.2%(±0.1%)。
- AC-02b:噪音點(label = −1)保留於主檔,不納入熱點統計。

**FR-03 時間序列化(M)**
- 描述:產出 (a) 全市日事故數序列(2021/01/01–2025/12/31,無事故日補 0);(b) 熱點×月事故數 panel(145×60);(c) 各熱點之 24hr/週/月分布向量。
- AC-03:日序列長度 = 1,826 且無缺日;panel 無缺月。

**FR-04 特徵萃取(M)**
- 描述:日序列特徵 = lag 1/7/14/28、7 與 28 日移動平均/標準差、星期幾(one-hot)、月份;分布向量 = 24+7+12 = 43 維,列正規化為比例。
- AC-04:特徵矩陣經暖機期截斷後無 NaN;特徵定義表輸出至 outputs/。

**FR-05 PCA 降維(M)**
- 描述:43 維分布向量標準化後 PCA;輸出主成分座標(2–3 維)、累積解釋變異、負荷量。
- AC-05:輸出累積解釋變異曲線;若前 3 成分 < 60% 須於報告討論。

**FR-06 K-means 熱點型態分群(M)**
- 描述:對 PCA 座標執行 K-means(random_state=42);以 elbow 與 silhouette 選 k(候選 2–8);輸出各群之平均 24hr 曲線並命名型態(如通勤尖峰型、夜間型、全日型)。
- AC-06:輸出 silhouette 分數表與各群曲線圖;每一熱點皆有群標籤。

**FR-07 預測建模與 80/20 驗證(M)**
- 描述:主模型 = 全市日事故數預測;baseline = Naive(lag-7)與 SARIMA(s=7,訓練期 AIC 選階);挑戰者 = XGBoost(全特徵);選用 LSTM(輸入窗 28 天)。輔助模型 = 熱點月事故數全域 XGBoost(過去 12 個月 + 靜態特徵)。切分:訓練 2021/01–2024/12、驗證 2025 全年;特徵僅用切分點以前資訊。
- AC-07a:輸出各模型於驗證期之 MAE、RMSE。
- AC-07b:挑戰者優於 Naive baseline,否則於報告說明原因。
- AC-07c:程式中不存在隨機切分或未來資訊洩漏(滯後特徵以 shift 實作)。

**FR-08 覆蓋缺口分析與建議清單(M)**
- 描述:haversine 計算各熱點至最近「測速」與最近「任何」設備之距離;篩選 300 m 內無任何設備之熱點;依五年事故數(第一階段)或預測風險(第二階段)排序;依 FR-06 型態對映建議設備類型(夜間型→測速照相;尖峰路口型→闖紅燈/科技執法)。
- AC-08:重現前 10 名建議清單(第 1 名:中正區羅斯福路4段與基隆路4段口,539 件,最近設備 915 m)。

**FR-09 視覺化(M)**
- 描述:產出 F1 密度+熱點圖、F2 熱點 vs 設備疊圖、F3 A1/A2 比例、F4 日序列(標注 COVID 期)、F5 PCA/K-means 散佈與群曲線、F6 預測 vs 實際;第二階段 F7 雨日 vs 非雨日箱型圖、F8 假日 vs 平日箱型圖、F9 特徵加入前後指標對照。
- AC-09:圖檔輸出至 figures/,PNG ≥ 150 dpi,含標題與軸標籤。

**FR-10 外部資料合併〔第二階段〕(S)**
- 描述:下載並解析 IN-7~IN-11;以日期為鍵 left join 至日序列;建立特徵:雨日、大雨日(>40 mm)、連續雨日、假日、補班日、連假第 n 天、COVID 警戒 dummy(2021/5/19–7/26)、捷運運量、夜間時數(astral 計算);速限與路口流量併入熱點靜態特徵。
- AC-10:合併後無日期缺漏;各外部欄位缺值率 < 1%;重訓 FR-07 模型並輸出加入前後之指標對照。

## 5. 附錄:已知限制(摘自報告資料文件第 8 節)

1. 無設置日期 → 僅相關性與風險預測,非因果估計。
2. 選址內生性 → 設備附近事故多 ≠ 設備無效。
3. 斑點圖無肇因欄位 → 以熱點型態分群間接推論,建議未來介接事故明細資料。
4. 未以車流量正規化 → 排序屬絕對風險。
5. A1 僅約 0.3% → 不以 A1 為直接預測目標,以 A1+A2 合計為目標、A1 作排序加權。
